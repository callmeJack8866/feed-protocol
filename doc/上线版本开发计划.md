#### P0-2 钱包认证 — EIP-4361 SIWE
**方案依据**: §16.2 安全措施 — 签名验证、EIP-712  
**当前状态**: 使用 `x-wallet-address` HTTP header（可被任意伪造）  
**工作量**: 3 天

```
工作内容:
1. 后端安装 siwe 库: npm install siwe ethers
2. 新建/重写 auth.controller.ts:
   - GET /api/auth/nonce → 生成随机 nonce 存 session
   - POST /api/auth/login → 接收 SIWE message + signature，验证后签发 JWT
   - POST /api/auth/logout → 清除 session
3. 新建 auth.middleware.ts:
   - 验证 JWT token（替代当前 x-wallet-address header）
   - 从 JWT 中提取 wallet address 注入 req.user
4. 全部 controller 迁移到 JWT 中间件保护
5. 前端 useWallet hook 集成 SIWE 签名流程:
   - 连接钱包后请求 nonce
   - 构造 SIWE message → wallet 签名
   - 发送到后端验证 → 获取 JWT → 存储
6. 前端所有 API 调用从 x-wallet-address 改为 Authorization: Bearer <JWT>
```

**涉及文件**:
- `feed-engine-backend/src/controllers/auth.controller.ts` [重写]
