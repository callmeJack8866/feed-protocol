#### 合约功能详解

**① FEEDToken — ERC-20 治理代币**
> 整个生态的"货币"

- 标准 ERC-20 代币（可转账、可授权）
- **质押保证金**：喂价员必须质押 FEED 才能接单（F级 100U → S级 25,000U）
- **奖励发放**：成功喂价后从奖励池分配 FEED
- **惩罚扣除**：恶意喂价扣除已质押的 FEED
- **DAO 治理**：持有 FEED 可参与仲裁投票

**② FeederLicense — ERC-721 喂价员执照 NFT**
> 喂价员的"上岗证"

- 注册时铸造一个 NFT 执照，作为链上身份凭证
- 记录等级（E/D/C/B/A/S 六级）、累计 XP、专业领域
- **Soulbound 特性**：不可转让，防止资格倒卖
- 赛季冠军额外获得特殊版 NFT 勋章

**③ FeedConsensus — 共识算法合约**
> 多人喂价取"最终价格"的裁判

- 接收 3-5 个喂价员对同一订单的价格提交
- **去极值中位数算法**：去掉最高最低，取中间值，防止操纵
- **偏差检测**：超过 1% 自动标记异常
- 共识结果写入链上，任何合约可查询

**④ RewardPenalty — 奖惩分配合约**
> 赏罚分明的"财务部"

- 奖励分配 **70/10/10/10**：
  - 🟢 70% → 喂价员 | 🟡 10% → 平台 | 🔵 10% → DAO 金库 | 🔴 10% → 销毁（通缩）
- 4 级惩罚体系：
  - MINOR → 警告 | MODERATE → 冻结 24h | SEVERE → 扣质押 + 冻结 7 天 | EXTREME → 永久封禁

**⑤ FeedEngine — 总控引擎合约**
> 整个系统的"大脑"，协调所有合约

- **订单创建**：接收外部协议的喂价请求
- **流程协调**：请求 → 分配喂价员 → 收集价格 → 共识计算 → 奖惩分配
- **NST 协议接入**：`requestFeed()` — 标准化外部喂价入口
- **共识查询**：`getConsensusPrice()` — 公开查询接口
- **白名单管理**：`setAuthorizedProtocol()` — 授权协议才能调用

**⑥ ArbitrationDAO — 链上仲裁合约**（治理层）
- 争议案件创建 → DAO 成员链上投票 → 裁决执行
- 超级多数才能推翻原裁决的申诉机制

#### 合约调用关系

```
外部协议 (如 NST Options)
    │
    ▼
FeedEngine.requestFeed()               ← 入口
    ├──→ FeederLicense.checkQualification()  ← 检查资格
    ├──→ FEEDToken.checkStake()              ← 检查质押
    │
    │  (喂价员提交价格)
    │
    ├──→ FeedConsensus.submitConsensus()      ← 计算共识
    │
    └──→ RewardPenalty.distributeRewards()    ← 发奖惩
              ├──→ FEEDToken.transfer()  (70% 喂价员)
              ├──→ FEEDToken.transfer()  (10% 平台)
              ├──→ FEEDToken.transfer()  (10% DAO)
              └──→ FEEDToken.burn()      (10% 销毁)
```

## 四、关键技术决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 数据库 | PostgreSQL（设计）+ SQLite（本地开发） | 开发便利性 vs 生产需求 |
| 认证方案 | EIP-4361 SIWE + JWT | Web3 原生 + 无状态认证 |
| 合约模式 | UUPS 可升级代理 | 支持后续升级 |
| 缓存方案 | Redis 优先 + 内存降级 | 高可用 + 开发独立性 |
| 状态管理 | Zustand | 轻量级，消除 props drilling |
| Hero 展示 | 纯 CSS 动画 | 避免图片透明度问题 |
